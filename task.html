1. Introduction

This tutorial describes the core components of the Robot Simulator by defining the basic structures and functioning of environment and robot. The use of these components will be illustrated in a toy example, i.e., a robot is randomly placed in the environment and moves to another location.

Important

The Robot Simulator does not have a comprehensive documentation. This tutorial gives hints on how to use it. However, there will be cases in which you may need checking the Robot Simulator code to understand the possibilities and its functioning. Therefore, you may need to inspect the Robot Simulator source code. 

git clone https://gitlab.emse.fr/maqit/simulation/robot_simulator.git

3. Java Project

3.1. Basic Robot Simulator Template

To use the predefined basic Java project to use the Robot Simulator

  • Unzip the compressed file

  • Run the project

    cd basic_simulator
        chmod 755 gradlew
        ./gradlew clean
        ./gradlew run

3.2. Manual Creation

To manually create a Java project to use the Robot Simulator

  • Clone the Robot Simulator git repository

    git clone https://gitlab.emse.fr/maqit/simulation/robot_simulator.git

  • Generate the simulator package

    cd robot_simulator
        mvn clean
        mvn package

  • Create a Java project

  • Import the maqit-simulator-1.0-jar-with-dependencies.jar file from the robot_simulator/target folder into the Java project

5. Create a Simulator

A simulation is an instance of a subclass of the SimFactory abstract class. To create your simulator subclass, you have

  • Extend the BasicSimulator class with the SimFactory class

  • Create all the abstract methods in the SimFactory class

  • Create a constructor that receives an instance of SimProperties as input parameter

The SimFactory class requires two Generic Types, GridEnvironment class and SituatedComponent class to be extended. In addition, the SimFactory has two attributes:

  • sp : Attribute initialized by the constructor, an instance of the SimProperties class. This attribute contains all configuration parameters of the simulation.

  • environment : Attribute initialized in the createEnvironment method, an instance of the GridManagement class. This attribute manages the environment components.

The Robot Simulator proposes different types of environments. Here, we are going to use the ColorGridEnvironment class. Therefore, you should add the following in the createEnvironment method. 

this.environment = new ColorGridEnvironment(this.sp.seed);

Additionally, you have to create a method to display the environment in the graphical window. Therefore, you should add the following methods in the BasicSimulator class. 

  public void initializeGW() {
            gwindow = new GraphicalWindow((ColorSimpleCell[][])(this.environment.getGrid()), this.sp.display_x, this.sp.display_y, this.sp.display_width, this.sp.display_height, this.sp.display_title);
            gwindow.init();
          }
        
          public void refreshGW() {
            gwindow.refresh();
          }

Now you can complete the main method by instantiating the BasicSimulator class, creating the environment and display the environment.

6. Environment

At this point, the grid is empty. You have to populate it with different components. In this section, you will learn how to add obstacles in the environment.

The basic process for creating an obstacle is

  1. to retrieve a free position in the environment grid. You can use the getPlace() method of the GridEnvironment class

  2. to create the obstacle in this position using the ColorObstacle class

  3. to add the obstacle to the list of components in the simulation by calling the addNewComponent method

  4. Update the simulation grid using the refreshGW method

While steps 1-3 are performed in the createObstacle method (you can implement a repeat logic to create all obstacles), step 4 is executed in the main method.

7. Robot

7.1. Robot Class

To create a robot in the simulation, the robot class has to extend the SituatedComponent class and implement the MobileComponent interface.

The inherited attributes

  • int id : identifier of the robot

  • int x, y : location of the robot in the environment

  • Cell[][] grid : the local perception

  • int rows : the maximum number of rows in the environment

  • int columns : the maximum number of columns in the environment

The constructor has the parameters

  • String name : the name of the robot

  • int field : the perception field of the robot

  • int debug : informs if in debug mode (1) or not (0)

  • int[] pos : the initial position of the robot in the environment

  • int[] rgb : the RGB color of the robot

  • int rows : the maximum number of rows in the environment

  • int columns : the maximum number of columns in the environment

The turn and move methods

  • turnLeft, turnRight : change the orientation of the robot

  • moveForward, moveBackward : move the robot

The perception is defined by the attributes int field (range of perception), Cell[][] grid, int rows and int columns (grid limits), and the updatePerception(Cell[][]) method that updates the values of the grid attribute.

7.2. Robot Behavior

The grid is a matrix of cells with n rows (x-axis) and m columns (y-axis). The content of a cell is an integer corresponding to the identifier of a situated component or -1 for a non-existent cell or 0 for an empty cell.

8. Simulation Execution

To run the simulation, you have to implement the schedule method. The basic principle is

  • Browse all the robots, for each robot

    • Retrieve the part of the grid corresponding to the perception of the robot

    • Update the perception of the robot

    • Make it act of a time step by calling the move(…​) method

  • Finally update the environment

  public void schedule(){
            List<Robot> lr = this.environment.getRobot();
            for(int i = 0; i < this.sp.step; i++){
              for(Robot r: lr) {
                int[] pos = r.getLocation();
                Cell[][] per = this.environment.getNeighbor(r.getX(), r.getY(), r.getField());
                r.updatePerception(per);
                r.move(1);
                updateEnvironment(pos, r.getLocation());
                }
                refreshGW();
              try {
                Thread.sleep(this.sp.waittime);
              } catch(InterruptedException ex) {
                System.out.println(ex);
              }
            }
          }

9. Robot Communication

The robots can send message to other robots in the environment.

The Message class has

  • Attributes : int emitter, int receiver, and String content

    Note
    		 If the receiver is set to 0, the message is broadcast.

  • Constructor : emitter and content ⇒ Broadcast by default

  • Methods : setters and getters

The InteractionComponent interface define the communication methods

  • sendMessage(Message) : add the message in an internal list of the sender

  • receiveMessage(Message) : add the message in an internal list of the receiver

  • readMessage() : read all messages and call handleMessage(Message) method
